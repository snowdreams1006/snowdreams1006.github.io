1. Claude Code 安装教程：3分钟上手，让AI帮你写代码

1.1. 一、这玩意儿是啥？

简单说，Claude Code 就是一个装在电脑里的AI程序员。

你跟它说话，它帮你：

  • ✅ 读代码、改代码
  • ✅ 查Bug、修Bug
  • ✅ 写新功能
  • ✅ 解释项目结构

不用复制粘贴到网页，直接在终端（黑框框）里聊，方便得很。

1.2. 二、安装前准备

1.2.1. 你需要：

  1. 一台Mac电脑（目前官方主要支持Mac，Windows可以用WSL）
  2. 网络能翻墙（因为Claude服务器在国外）
  3. 5分钟时间

1.2.2. 检查网络代理

如果你用Clash、V2Ray这类工具，先确认：

  • 软件是打开状态
  • 记住端口号（一般是7890，像Clash默认就是这个）

1.3. 三、开始安装（就一行命令）

打开终端（按Command+空格，搜"终端"），粘贴下面这串：

curl -fsSL https://claude.ai/install.sh | bash

如果网络不通，先设置代理再运行：

# 设置代理（把7890换成你的实际端口）
export http_proxy=http://127.0.0.1:7890 
export https_proxy=http://127.0.0.1:7890 

# 再执行安装
curl -fsSL https://claude.ai/install.sh | bash

看到下面这样，就是装好了：

✔ Claude Code successfully installed!        
Version: 2.1.71
Location: ~/.local/bin/claude

1.4. 四、配置环境变量（重要！）

装好了但找不到命令？因为系统不知道它装在哪了。

复制这行，粘贴运行：

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

如果你用的是bash（比较少见），把.zshrc换成.bashrc

验证一下：

claude --version

能看到版本号，说明成功了！

1.5. 五、登录认证（二选一）

Claude Code要连服务器，得证明你是谁。两种方式：

方式A：订阅会员（适合天天用）

  • 去Claude官网买Pro或Max会员
  • 运行claude，会弹出浏览器让你登录
  • 适合：每天写代码的同学

方式B：API Key（适合偶尔用）

  • 按量付费，用多少扣多少
  • 在Anthropic Console申请Key
  • 设置环境变量：
export ANTHROPIC_API_KEY="你的key"

省钱小技巧：国内用户可以用火山引擎的"方舟Coding Plan"，更便宜，支持DeepSeek、Kimi等国产模型。

1.6. 六、开始用！4个常用命令

  1. 进入项目，启动对话
cd 你的项目文件夹
claude

就像微信聊天一样，你打字，它回复。

  1. 问个问题，答完就撤
claude -p "帮我看看这个文件夹是干嘛的"

适合快速问一嘴，不用进聊天界面。

  1. 接着上次聊
claude -c

昨天聊到一半，今天继续。

  1. 回到之前的状态
claude -r

就像游戏读档，回到某个检查点。

1.7. 七、国内用户专属：接入国产大模型

不想用Claude官方？想省钱？用火山引擎的方舟计划：

  • 第一步：创建配置文件
vim ~/.claude/settings.json

粘贴进去（把<ARK_API_KEY>换成你的真实Key）：

{
    "env": {
        "ANTHROPIC_AUTH_TOKEN": "你的ARK_API_KEY",
        "ANTHROPIC_BASE_URL": "https://ark.cn-beijing.volces.com/api/coding",
        "ANTHROPIC_MODEL": "kimi-k2.5"
    }
}
  • 第二步：跳过新手引导
vim ~/.claude.json

写入：

{
  "hasCompletedOnboarding": true
}
  • 第三步：启动使用
claude

想换模型？ 在对话里输入：

/model deepseek-v3.2

支持的模型有：kimi-k2.5、deepseek-v3.2、doubao-seed-2.0-pro等。

1.8. 八、遇到问题怎么办？

❌ 提示"command not found: compdef" 解决：

# 检查文件在不在
ls -l ~/.local/bin/claude

# 给执行权限
chmod +x ~/.local/bin/claude

# 重新加载配置
source ~/.zshrc

❌ 提示"Unable to connect" 解决：

claude
/login

重新登录一遍。
